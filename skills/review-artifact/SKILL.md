# Ревью артефакта

Ты открываешь markdown-артефакт в браузерном редакторе, собираешь аннотации пользователя и применяешь правки.

---

## Вход

`$ARGUMENTS` — путь к markdown-файлу (task, plan или report).

Если `$ARGUMENTS` пуст — спроси путь через AskUserQuestion:

> Укажи путь к файлу для ревью (task, plan или report).

Варианты:

- **Указать путь** — пользователь вводит путь
- **Отменить** — выход

Проверь, что файл существует. Если нет — сообщи и выйди.

---

## Проверка binary

Перед запуском проверь наличие binary:

```
${CLAUDE_PLUGIN_ROOT}/packages/annotator/bin/sp-annotator
```

Если файл не существует или не исполняемый — сообщи пользователю:

> sp-annotator не собран. Выполните: cd packages/annotator && bun install && bun run build

Выйди.

---

## Цикл ревью

### Шаг 1 — Запуск annotator

Вызови через Bash tool:

```bash
${CLAUDE_PLUGIN_ROOT}/packages/annotator/bin/sp-annotator review <path>
```

Где `<path>` — абсолютный путь к markdown-файлу.

Annotator откроет браузер, покажет файл с возможностью аннотирования. Когда пользователь нажмёт Approve или Submit Feedback — annotator завершится и выведет JSON в stdout.

### Шаг 2 — Парсинг результата

Парси JSON stdout. Формат:

```json
{
  "action": "approved" | "feedback",
  "annotations": [
    {
      "anchorText": "текст из документа",
      "comment": "комментарий пользователя",
      "suggestion": "предложенная замена (опционально)"
    }
  ]
}
```

### Шаг 3 — Обработка action

**При `action = "approved"`:**

Сообщи:

> Артефакт одобрен.

Верни управление вызывающему скиллу. Выход.

**При `action = "feedback"`:**

Для каждой аннотации из массива `annotations`:

1. Найди `anchorText` в файле
2. Если есть `suggestion` — замени `anchorText` на `suggestion` через Edit tool
3. Если только `comment` без `suggestion` — интерпретируй комментарий и внеси правку через Edit tool

После применения всех аннотаций предложи повторное ревью через AskUserQuestion:

> Аннотации применены (N шт). Что дальше?

Варианты:

- **Повторное ревью (Recommended)** — вернись к Шагу 1
- **Завершить** — выход

---

## Правила

- **Без остановок.** Применяй все аннотации за один проход, не спрашивая подтверждение на каждую.
- **Точные правки.** Используй Edit tool для каждой замены, не перезаписывай весь файл.
- **Цикл до approve.** Повторяй ревью столько раз, сколько нужно, пока пользователь не одобрит или не завершит вручную.
- **Язык контента** — русский.
