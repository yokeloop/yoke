## Что делать

Проверь каждый пункт. Обновляй только то что релевантно.

### 1. README

Прочитай README проекта. Обнови если:
- Добавлен новый публичный API endpoint → добавь в описание API
- Добавлена новая фича → добавь в описание
- Изменился setup или конфигурация → обнови инструкцию
- Добавлена новая зависимость → обнови prerequisites

Если README не нуждается в обновлении — не трогай.

### 2. CHANGELOG

Если в проекте есть CHANGELOG.md:
- Добавь запись в секцию Unreleased (или текущую версию)
- Формат: следуй существующему формату файла

Если CHANGELOG нет — не создавай.

### 3. JSDoc / TSDoc

Для каждого нового или изменённого экспортируемого символа
(функция, класс, тип, интерфейс):
- Добавь или обнови JSDoc/TSDoc
- Опиши: что делает, параметры, возвращаемое значение
- Следуй существующему стилю документации в проекте

Не документируй приватные / внутренние функции.

## Что НЕ делать

- Не создавать документацию с нуля если её не было
- Не создавать README если его нет
- Не трогать файлы за пределами проекта
- Не добавлять избыточную документацию
- Не менять код — только документацию и комментарии

- src/auth/forgot-password.ts: добавил JSDoc для forgotPassword()
