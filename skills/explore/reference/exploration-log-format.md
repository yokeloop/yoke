# Exploration Log Format

Формат файла `<slug>-exploration.md` и Q&A записей.

## Header файла (при создании)

```markdown
# Exploration: {{TOPIC}}

**Дата:** {{YYYY-MM-DD}}
**Вопросов:** {{N}}
```

---

## Формат Q&A записи

```markdown
## Q{{N}}: {{вопрос}}

{{Prose-ответ с объяснением — 2-5 предложений. Можно ссылаться на конкретные места в коде.}}

**Key files:**

- `{{path/to/file.ts}}:{{LINE}}` — {{краткое описание}}
- `{{path/to/other.ts}}:{{LINE}}` — {{краткое описание}}

**Sources:**

- {{ссылка или упоминание источника}}
```

---

## Формат brainstorm записи

Brainstorm-запись дополнительно содержит секцию `### Варианты`.

```markdown
## Q{{N}}: {{вопрос}} (brainstorm)

{{Prose с контекстом задачи — почему нужен выбор.}}

**Key files:**

- `{{path/to/file.ts}}:{{LINE}}` — {{краткое описание}}

### Варианты

- [ ] **{{Метка варианта}}** — {{описание, плюсы/минусы}}
- [ ] **{{Метка варианта}}** — {{описание, плюсы/минусы}}
- [ ] **Other:** \_\_\_

**Sources:**

- {{ссылка или упоминание источника}}
```

---

## Формат итогового summary

```markdown
## Summary

{{3-5 предложений. Тема, ключевые выводы, принятые и отложенные решения.}}
```

---

## Правила

- Нумеруй Q с 1, строго последовательно.
- Используй формат даты `YYYY-MM-DD`.
- Не используй emoji.
- Добавляй секцию `**Sources:**`, если есть источники.
- Добавляй секцию `### Варианты` только в brainstorm-записях.
- Добавляй секцию `## Summary` в конце, после всех Q&A записей.
