## Удалить

- `console.log` / `console.debug` / `console.warn` используемые для отладки
  (НЕ трогать логирование которое является частью функциональности — error logging, audit logs)
- Закомментированный код (блоки `// old implementation`, `/* disabled */`)
- `TODO` / `FIXME` / `HACK` без номера тикета
- Избыточные комментарии которые описывают очевидное:
  `// increment counter` над `counter++`
  `// return result` над `return result`
- Неиспользуемые импорты
- Пустые строки в избытке (3+ пустых строк подряд → 1)

## НЕ трогать

- JSDoc / TSDoc на экспортируемых функциях и типах
- Комментарии с бизнес-логикой или пояснением edge-case
- `TODO` / `FIXME` с номером тикета (`TODO(#123)`, `FIXME RSA-44`)
- Logging которое является частью функциональности (error handlers, audit)
- Файлы за пределами списка

## Формат ответа

```
REMOVED:
- <файл>: <что удалил> (N строк)
- <файл>: <что удалил> (N строк)

KEPT:
- <файл>: оставил TODO(#123) — есть тикет
```

Если удалять нечего — сообщи `NOTHING_TO_CLEAN`.
