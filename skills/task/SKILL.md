# Создание задачи из тикета

Создаёт markdown-файл задачи из тикета в трекере. Принимает URL тикета и текст/описание задачи через `$ARGUMENTS`.

## Вход

`$ARGUMENTS` содержит:

1. **URL тикета** — ссылка на issue (GitHub Issues, YouTrack и др.)
2. **Текст тикета** — заголовок и описание задачи (вставлен целиком или пересказан)

Если пользователь передал только URL без текста — попробуй получить содержимое тикета:
- Для GitHub Issues: используй `gh issue view <url>` через Bash
- Для других трекеров: попроси пользователя вставить текст тикета

## Формирование task-slug

`task-slug` = идентификатор тикета + краткое описание на английском в kebab-case.

### Извлечение идентификатора

| Трекер | URL | Идентификатор |
|--------|-----|---------------|
| GitHub Issues | `https://github.com/owner/repo/issues/86` | `86` |
| YouTrack | `https://tracker.example.com/issue/R2-50/...` | `R2-50` |
| Другие | любой URL с числовым или буквенно-числовым ID | извлеки ID из URL |

### Генерация описания

Из заголовка/текста тикета сформируй краткое описание на английском (2-4 слова, kebab-case).

### Примеры

| URL | Заголовок тикета | task-slug |
|-----|-------------------|-----------|
| `.../issues/86` | Сделать страницу блэкджека | `86-black-jack-page` |
| `.../issues/87` | Добавить robots.txt | `87-robots-txt` |
| `.../issue/R2-50/...` | Сделать запись очков в БД User ID | `R2-50-user-id-to-db` |
| `.../issue/R2-102/...` | Обновить документацию по 3 играм | `R2-102-doc-update` |
| `.../issue/R2-9/...` | Сделать админку для Leaderboard | `R2-9-leaderboard-admin` |

## Выход

Файл: `docs/ai/<task-slug>/<task-slug>.md`

### Шаблон файла задачи

```markdown
# <Заголовок задачи>

**Тикет:** <URL тикета>

## Описание

<Что нужно сделать — переработай текст тикета в чёткие требования. Не копируй дословно, а структурируй и уточни.>

## Критерии готовности

- <Конкретная проверка 1>
- <Конкретная проверка 2>
- <...>
```

## Процесс

### Шаг 1: Разбор входных данных

1. Извлеки URL тикета из `$ARGUMENTS`
2. Извлеки идентификатор тикета из URL
3. Определи текст тикета — из `$ARGUMENTS` или запроси через `gh issue view` (для GitHub)

### Шаг 2: Формирование task-slug

1. Возьми идентификатор тикета (число или ID вида `R2-50`)
2. Из заголовка/описания сформируй краткое описание на английском в kebab-case
3. Собери `task-slug`: `<идентификатор>-<описание>`

### Шаг 3: Создание файла

1. Создай директорию `docs/ai/<task-slug>/` через Bash: `mkdir -p docs/ai/<task-slug>`
2. Запиши файл `docs/ai/<task-slug>/<task-slug>.md` через Write, используя шаблон выше
3. Заполни:
   - **Заголовок** — краткая формулировка задачи на русском
   - **URL тикета** — оригинальная ссылка
   - **Описание** — переработай текст тикета в структурированные требования
   - **Критерии готовности** — конкретные проверяемые условия завершения

### Шаг 4: Подтверждение

Сообщи пользователю:
- Путь к созданному файлу
- Сформированный task-slug

## Правила

- Описание в файле задачи — на русском
- task-slug — на английском в kebab-case
- Не копируй текст тикета дословно — переработай в чёткие требования
- Критерии готовности должны быть конкретными и проверяемыми
- Если текст тикета неясен — задай уточняющие вопросы перед созданием файла
