---
# Exploration Log Format

Формат файла `<slug>-exploration.md` и отдельных Q&A записей.
---

## Header файла (при создании)

```markdown
# Exploration: {{TOPIC}}

**Дата:** {{YYYY-MM-DD}}
**Вопросов:** {{N}}
```

---

## Формат Q&A записи

```markdown
## Q{{N}}: {{вопрос}}

{{Prose-ответ с объяснением — 2-5 предложений. Можно ссылаться на конкретные места в коде.}}

**Key files:**

- `{{path/to/file.ts}}:{{LINE}}` — {{краткое описание}}
- `{{path/to/other.ts}}:{{LINE}}` — {{краткое описание}}

**Sources:**

- {{ссылка или упоминание источника}}
```

---

## Формат brainstorm записи

Brainstorm-запись — это Q&A запись с добавленной секцией `### Варианты`.

```markdown
## Q{{N}}: {{вопрос}} (brainstorm)

{{Prose с контекстом задачи — почему нужен выбор.}}

**Key files:**

- `{{path/to/file.ts}}:{{LINE}}` — {{краткое описание}}

### Варианты

- [ ] **{{Метка варианта}}** — {{описание, плюсы/минусы}}
- [ ] **{{Метка варианта}}** — {{описание, плюсы/минусы}}
- [ ] **Other:** \_\_\_

**Sources:**

- {{ссылка или упоминание источника}}
```

---

## Формат итогового summary

```markdown
## Summary

{{3-5 предложений. Что исследовалось, какие ключевые выводы сделаны, какие решения приняты или отложены.}}
```

---

## Правила

- Нумерация Q начинается с 1 и строго последовательная.
- Дата: формат `YYYY-MM-DD`.
- Без emoji.
- Секция `**Sources:**` — только при наличии источников. Без источников секцию опусти.
- Секция `### Варианты` — только в brainstorm-записях. В обычных Q&A секцию опусти.
- Секция `## Summary` — итоговая, добавляется в конце после всех Q&A записей.
