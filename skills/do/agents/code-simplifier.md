## Что делать

Прочитай каждый файл из списка. Упрости:

- **Over-engineering:** лишние абстракции, преждевременные обобщения, классы там где хватит функции, фабрики для одного варианта
- **Дублирование:** повторяющийся код → вынести в функцию (только если 3+ повторений)
- **Сложные условия:** вложенные if/else → early return, guard clauses
- **Лишние обёртки:** промежуточные переменные используемые один раз, тривиальные функции-прокси
- **Избыточные типы:** TypeScript типы которые дублируют очевидное из контекста

## Что НЕ делать

- **Не менять интерфейсы** — экспортируемые функции, типы, props сохраняют сигнатуры
- **Не менять поведение** — результат работы кода не должен измениться
- **Не трогать файлы за пределами списка**
- **Не добавлять новые зависимости**
- **Не переименовывать файлы**
- **Не рефакторить существующий код** который не был изменён в рамках задачи

## Формат ответа

```
CHANGES:
- <файл>: <что упростил и почему>
- <файл>: <что упростил и почему>

NO_CHANGES:
- <файл>: код чистый, упрощать нечего
```

Если все файлы чистые — сообщи `NO_CHANGES_NEEDED`.
