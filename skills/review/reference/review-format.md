# Review Report Format

Формат выходного файла `<SLUG>-review.md`. Записывается агентом review-report-writer.

---

## Шаблон

```markdown
# Code Review: <SLUG>

## Summary

### Контекст и цель

<1-3 предложения: что сделано и зачем>

### Ключевые участки для ревью

1. **`src/path/file.ts:fn()`** — <почему важен>

### Сложные решения

1. **<Что>** (`src/path/file.ts:42`)
   <Почему так сделано. Trade-off'ы.>

### Вопросы к ревьюеру

1. <Конкретный вопрос по архитектуре, контрактам, перформансу, безопасности или читаемости>

### Риски и влияние

- <Риск>: <на что обратить внимание>

### Тесты и ручные проверки

**Авто-тесты:**

- <что должно быть покрыто>

**Ручные сценарии:**

1. <Шаг> → <ожидаемый результат>

### Out of scope

- <что оставлено за рамками PR намеренно>

## Commits

| Hash    | Описание        |
| ------- | --------------- |
| abc1234 | feat(slug): ... |

## Changed Files

| Файл             | +/-     | Описание         |
| ---------------- | ------- | ---------------- |
| src/path/file.ts | +42/-10 | <что изменилось> |

## Issues Found

| Severity | Score | Category    | File:line            | Description             |
| -------- | ----- | ----------- | -------------------- | ----------------------- |
| high     | 8     | security    | src/auth/login.ts:42 | SQL injection в запросе |
| medium   | 5     | performance | src/api/list.ts:15   | N+1 запрос в цикле      |
| low      | 2     | style       | src/utils/fmt.ts:8   | Неиспользуемый импорт   |

> Если проблем не найдено — заменить таблицу текстом: **Проблем не найдено.**

## Fixed Issues

| Issue                       | Commit    | Description                  |
| --------------------------- | --------- | ---------------------------- |
| SQL injection в login.ts:42 | `def5678` | Параметризованный запрос     |
| N+1 запрос в list.ts:15     | `ghi9012` | Batch-загрузка через Promise |

> Если фиксов не было — заменить таблицу текстом: **Фиксов не потребовалось.**

## Skipped Issues

| Issue                 | Reason                        |
| --------------------- | ----------------------------- |
| Неиспользуемый импорт | Стилистическое, вне скоупа PR |

> Если пропущенных нет — заменить таблицу текстом: **Все найденные проблемы исправлены.**

## Recommendations

- <Рекомендация для PR review: на что обратить внимание при мердже>
```

---

## Правила

- **Summary** — 7 измерений: контекст и цель, ключевые участки, сложные решения, вопросы к ревьюеру, риски, тесты, out of scope.
- **Issues Found** — сортировка по Score убыванием. Severity: `high` / `medium` / `low`.
- **Fixed Issues** и **Skipped Issues** — только при наличии Issues Found.
- Пустые таблицы заменять текстовыми заглушками (см. блоки `>`).
- Язык — русский. Активный залог, конкретные формулировки, без лишних слов.
